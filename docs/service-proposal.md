**"요일, 건물, 시간을 클릭하면 공강 중인 강의실을 바로 보여주는 서비스"**

## 2. 문제 정의

### 2.1 도서관 현황 분석

**학교 규모 대비 도서관 규모의 한계:**

- **학생 수**: 약 9,000명
- **도서관 면적**: 3,162평 (약 10,446㎡)
- **1인당 면적**: 약 1.16㎡

**실제 좌석 수 현황:**

도서관 층별 좌석 현황 (서울여대 홈페이지 기준):
- 1층: 457좌석 (2,246.32㎡)
- 2층: 168좌석 (1,547.04㎡)
- 3층: 159좌석 (1,393.84㎡)
- 4층: 390좌석 (1,489.21㎡)
- 5층: 448좌석 (1,489.21㎡)
- 지하1층: 별도 (2,280.10㎡)
- **총 좌석: 1,622석**
- **1인당 좌석 비율: 약 0.18석/인** (권장 기준: 0.3-0.4석/인)

**결론**: 도서관의 물리적 규모가 학생 수 대비 부족합니다.

### 2.2 공강 시간대 좌석 부족 문제

**모든 학생이 동시에 도서관을 사용하지는 않지만, 특정 시간대에 집중:**

- **공강 시간대**: 점심시간(12:00-15:00), 오후 공강 시간
- 이 시간대에 도서관 이용 수요 급증
- 학생들이 공부할 공간을 찾기 위해 도서관을 방문

**불편한 점:**
- 자리가 없어 돌아다니는 시간이 길어짐
- 도서관을 찾아다니다가 결국 갈 곳이 없어 공강 시간을 낭비하게 됨

**문제점:**
- 서울여대 수업 시간 특성상 한 수업이 3시간이기 때문에 공강이 길어서 있을 장소가 필요함
- 긴 공강 시간 동안 안정적으로 공부할 수 있는 공간 부족

### 2.3 미활용 자원: 빈 강의실

**전체 강의실 현황:**

**건물별 면적** (서울여대 대학소개 연혁 기준) **및 예상 강의실 개수**:

| 건물 | 면적 | 예상 강의실 수 |
|------|------|--------------|
| 50주년기념관 | 22,453㎡ | 약 59개 |
| 인문사회관 | 17,804.59㎡ | 약 46개 |
| 조형예술관 | 9,382.61㎡ | 약 24개 |
| 제1과학관 | 8,517.10㎡ | 약 22개 |
| 제2과학관 | 9,380.31㎡ | 약 24개 |
| **총계** | **67,537.61㎡** | **약 175개** |

- **학교 보유 강의실**: 5개 건물, 총 약 175개 강의실
- **공강 시간대**: 일부 강의실은 수업이 없음
- **현재 상황**: 이 빈 강의실을 활용할 정보가 없음

**문제점:**
- 도서관은 포화 상태 ↔ 강의실은 비어있음
- 학생들은 강의실이 비어있는지 알 수 없음
- **결과**: 학교 시설이 제대로 활용되지 않음

### 2.4 해결해야 할 과제

1. **공강 시간대 학습 공간 부족** → 빈 강의실 정보 제공
2. **정보 접근의 어려움** → 간편한 검색 시스템 필요
3. **자원 활용 효율성** → 빈 강의실 활용도 증대

### 2.5 기존 시스템의 불편함

**학교 종합정보시스템을 통한 공강교실 검색의 현실:**

학생들이 현재 공강교실을 찾기 위해 거치는 과정:

1. 학교 종합정보시스템 접속
2. **로그인** (인증 필요)
3. 메뉴 탐색
4. 상세 조건 입력 (학부, 학년, 학과 등)
5. 추가 필터 선택
6. 시간표 조회
7. 결과 확인
8. 강의실 상황 파악

**문제점:**
- **복잡성**: 총 **최소 8번 이상의 클릭** 필요
- **로그인 필요**: 매번 인증 거쳐야 함
- **상세 조건 입력**: 학부, 학년, 학과 등 추가 정보 입력 필요
- **시간 소비**: 공강 시간을 사용해서 강의실을 찾아야 함
- **직관성 부족**: 결과 확인이 복잡하고 직관적이지 않음

## 3. 해결 방안

### 3.1 빈강의실 서비스

**간편한 3단계 검색으로 빈 강의실을 즉시 제공:**

1. **건물 선택** (5개 건물)
2. **요일 선택** (월~금)
3. **시간 선택** (1교시, 2교시, 3교시, 4교시, 5교시, 6교시)

**검색 결과:**
- 공강 중인 강의실 목록 즉시 표시
- 강의실명, 층, 시설 정보 제공

#### 기존 시스템 vs 빈강의실 비교

| 항목 | 기존 시스템 (학교 종합정보시스템) | 빈강의실 |
|------|--------------------------------|---------|
| **로그인** | ✓ 필수 | ✗ 불필요 |
| **클릭 횟수** | **최소 8회 이상** | **3회** |
| **조건 입력** | 복잡함 (학부, 학년, 학과 등) | 간단함 (건물, 요일, 시간) |
| **검색 속도** | 느림 (시스템 로드 포함) | 즉각적 |
| **사용 시기** | 사전에 미리 확인 필요 | 공강 시간에 바로 확인 |
| **접근성** | 학교 계정 필요 | 누구나 접속 가능 |
| **직관성** | 낮음 (복잡한 인터페이스) | 높음 (3가지 선택만으로 완료) |

**핵심 차이점:**
- 기존 시스템: 로그인 후 **최소 8번 이상의 클릭**과 상세 조건 입력 필요
- **빈강의실**: 지금 바로 **3번의 클릭(건물 → 요일 → 시간)**만으로 빈 강의실을 찾을 수 있음

### 3.2 서비스의 가치

- **공강 시간 활용**: 공부할 공간이 있어 공강 시간을 낭비하지 않음
- **팀 작업 공간 확보**: 조별 과제를 위한 모임 장소 확보
- **정보 접근 용이**: 복잡한 절차 없이 즉시 정보 제공
- **학교 자원 활용**: 비어있는 강의실이 제대로 활용됨

## 5. 핵심 기능 (MVP)

### 5.1 빈 강의실 검색

**검색 과정:**

1. **건물 선택**
   - 50주년 기념관
   - 제1과학관
   - 제2과학관
   - 인문사회관
   - 예술관

2. **요일 선택** 
   - 월요일 ~ 금요일 (요일 버튼 클릭)

3. **시간 선택**
   - 1교시 (09:00-10:00)
   - 2교시 (10:00-11:00)
   - 3교시 (11:00-12:00)
   - 4교시 (12:00-13:00)
   - 5교시 (13:00-14:00)
   - 6교시 (14:00-15:00)

**검색 결과 표시:**

선택된 조건에 맞는 빈 강의실 목록 즉시 표시:
- 강의실명 (예: 제2과학관 301호)
- 층 정보 (예: 3층)
- 시설 정보 (빔프로젝터, 화이트보드, 컴퓨터 등)

#### 기존 시스템과의 간편함 비교

**기존 시스템의 번거로움:**
- 로그인 필수
- 복잡한 메뉴 탐색
- 학부, 학년, 학과 등 상세 조건 입력
- 8회 이상의 클릭
- 공강 시간을 낭비하며 검색

**빈강의실의 간편함:**
- ✓ 로그인 불필요
- ✓ 바로 접속해서 바로 검색
- ✓ **단 3가지만 선택 (건물 → 요일 → 시간)**
- ✓ **3번의 클릭으로 완료**
- ✓ 지금 당장 빈 강의실 확인 가능

**결론:** 공강이 생기는 그 순간, 지금 바로 빈 강의실을 찾을 수 있는 서비스입니다.

### 5.2 특징

- **간결성**: 3개 선택지만으로 완료
- **속도**: 검색 결과를 즉시 확인
- **정확성**: 학교 공식 시간표 기반
- **접근성**: 로그인 없이 누구나 사용 가능

## 6. 화면 구성 개념

### 6.1 메인 검색 화면

**구성 요소:**
- 서비스 제목 (🏫 빈강의실)
- 부제 ("공강교실을 빠르게 찾아보세요")
- **검색 영역:**
  - 건물 선택 버튼들 (5개)
  - 요일 선택 버튼들 (월~금)
  - 시간 선택 버튼들 (1교시, 2교시, 3교시, 4교시, 5교시, 6교시)
  - 검색 버튼

**인터랙션:**
- 건물, 요일, 시간을 모두 선택하면 검색 버튼 활성화
- 검색 버튼 클릭 시 결과 화면으로 이동

### 6.2 검색 결과 화면

**구성 요소:**
- 검색 조건 요약 (예: "제2과학관 | 월요일 1교시")
- 빈 강의실 목록:
  - 각 강의실을 카드 형태로 표시
  - 강의실명 (건물 + 호수)
  - 층 정보
  - 시설 아이콘 (빔프로젝터, 화이트보드 등)

**빈 결과 상황:**
- 조건에 맞는 강의실이 없을 경우
- "빈 강의실이 없습니다" 메시지 표시
- 다른 요일이나 시간 선택 제안
