> **문서 버전**: 1.0  
> **작성일**: 2026년 1월 21일  
> **문서 유형**: 서비스 기획안 (MVP)

## 1. 서비스 개요

### 1.1 서비스명 제안

다음 중 하나를 선택하거나 참고하여 최종 서비스명을 결정할 수 있습니다:

1. **빈강** (Binkang)
   - 짧고 직관적인 한글 네이밍
   - "빈 강의실"의 줄임말로 서비스를 바로 이해 가능

2. **공강지도** (Gonggang Map)
   - 공강 시간에 교실을 찾는다는 의미
   - "지도"를 통해 교실을 찾는 서비스임을 명확히 전달

3. **ClassFinder** (클래스파인더)
   - 글로벌한 느낌의 영문 네이밍
   - 교실을 찾는다는 명확한 의미 전달

4. **빈자리** (Empty Seat)
   - 친근하고 캐주얼한 느낌
   - 학생들이 편하게 부를 수 있는 이름

### 1.2 서비스 한 줄 소개

**"요일과 시간을 입력하면 학교 내 사용 가능한 빈 강의실을 한눈에 찾아주는 서비스"**

### 1.3 배경 및 문제점

대학생들은 다음과 같은 상황에서 빈 강의실을 찾는 데 어려움을 겪고 있습니다:

#### 주요 불편사항
- **조별 과제 공간 부족**: 팀 프로젝트나 스터디를 위한 공간을 찾기 위해 건물을 돌아다니며 빈 교실을 찾아야 함
- **정보 부재**: 어느 교실이 어느 시간에 비어있는지 알 수 없어 매번 시행착오를 겪음
- **시간 낭비**: 빈 교실을 찾다가 공강 시간이나 수업 전 시간을 허비하게 됨
- **불확실성**: 겨우 찾은 교실도 곧 수업이 시작될 수 있다는 불안감
- **분산된 정보**: 학교 홈페이지나 포털에서 시간표를 일일이 확인해야 하는 번거로움

### 1.4 해결 방안

본 서비스는 다음과 같은 가치를 제공합니다:

- **즉시성**: 원하는 요일과 시간을 입력하면 즉시 사용 가능한 교실 목록 제공
- **편리성**: 모바일과 웹에서 언제 어디서나 접근 가능
- **정확성**: 학교 시간표 데이터를 기반으로 한 신뢰할 수 있는 정보
- **효율성**: 건물, 시설 등의 조건으로 필터링하여 원하는 조건의 교실만 선별
- **개인화**: 자주 사용하는 교실을 즐겨찾기로 저장하여 빠른 재접근

## 3. 핵심 기능 (MVP 범위)

### 3.1 필수 기능

#### 기능 1: 공강교실 검색

**목적**: 사용자가 원하는 시간에 비어있는 교실을 찾을 수 있도록 함

**주요 요소**:
- **요일 선택**: 월요일부터 일요일까지 선택 가능
- **시간 선택**: 두 가지 방식 지원
  - 교시제 방식: 1교시, 2교시, 3교시... (예: 9시, 10시, 11시)
  - 시간대 방식: 09:00-10:00, 10:00-11:00... 형태의 시간 범위
- **검색 버튼**: 조건을 입력하고 검색 실행

**검색 결과 표시**:
- 교실명 (예: 공학관 301호)
- 건물명 및 층수
- 수용 인원
- 보유 시설 아이콘 (빔프로젝터, 컴퓨터 등)
- 사용 가능 시간 (현재 시간부터 다음 수업 시작까지)
- 교실 유형 (강의실, 실습실, 세미나실 등)

**특징**:
- 간단하고 직관적인 UI로 3단계 이내에 검색 완료
- 검색 결과는 건물별, 거리순 등으로 정렬 가능

#### 기능 2: 필터링

**목적**: 검색 결과에서 사용자의 필요에 맞는 교실만 선별

**필터 옵션**:

1. **건물별 필터**
   - 체크박스 형태로 복수 선택 가능
   - 예: 공학관, 인문관, 경영관, 자연과학관 등
   - 사용자가 선호하는 건물의 교실만 표시

2. **시설별 필터**
   - 필요한 시설을 갖춘 교실만 표시
   - 옵션 예시:
     - 빔프로젝터 있음
     - 컴퓨터 있음
     - 화이트보드 있음
     - 에어컨 있음
     - 마이크/스피커 있음

3. **교실 유형별 필터**
   - 교실의 용도나 형태로 분류
   - 옵션 예시:
     - 강의실 (일반 강의실)
     - 실습실 (컴퓨터실, 실험실 등)
     - 세미나실 (소규모 토론실)
     - 대형 강의실 (100인 이상)

**특징**:
- 여러 필터를 동시에 적용 가능
- 필터 적용 시 실시간으로 결과 업데이트
- 적용된 필터 내역을 한눈에 확인 가능
- 필터 초기화 버튼으로 한 번에 해제

#### 기능 3: 즐겨찾기

**목적**: 자주 찾는 교실을 저장하여 빠르게 재접근

**주요 기능**:
- **즐겨찾기 추가**: 교실 상세 화면이나 검색 결과에서 별 아이콘 클릭
- **즐겨찾기 목록**: 저장한 교실들을 한 곳에 모아서 볼 수 있는 별도 탭
- **즐겨찾기 삭제**: 더 이상 필요하지 않은 교실 제거
- **빠른 접근**: 메인 화면에서 즐겨찾기한 교실의 현재 상태 확인

**저장 방식**:
- 로그인/회원가입 없이 사용자 기기에 저장
- 기기별로 개별 관리 (동기화는 향후 기능)

**특징**:
- 간편한 원클릭 저장
- 별도 인증 절차 불필요
- 즐겨찾기 순서 변경 가능 (드래그 앤 드롭)

### 3.2 부가 기능

#### 교실 상세 정보
- 교실 사진 (선택사항)
- 좌석 배치 형태 (예: 계단식, 평면식, U자형)
- 층별 위치 안내 텍스트
- 주변 편의시설 정보 (화장실, 엘리베이터, 자판기 위치 등)

#### 검색 기록
- 최근 검색한 요일/시간 조합 저장
- 빠른 재검색 지원

## 4. 사용자 시나리오

### 시나리오 1: 조별 과제를 위한 회의 공간 찾기

**상황**: 
민수는 5명의 팀원과 함께 다음 주 목요일 오후 2시부터 4시까지 조별 과제 회의를 하려고 합니다. 발표 준비를 위해 빔프로젝터가 있는 교실이 필요합니다.

**사용자 플로우**:

1. **앱/웹 접속**: 민수가 서비스에 접속합니다
2. **검색 조건 입력**:
   - 요일: 목요일 선택
   - 시간: 6교시(14:00-16:00) 선택
3. **검색 실행**: 검색 버튼 클릭
4. **필터 적용**:
   - 시설 필터에서 "빔프로젝터 있음" 체크
   - 교실 유형에서 "세미나실" 또는 "강의실" 선택
5. **결과 확인**: 조건에 맞는 교실 목록 표시 (예: 공학관 501호, 인문관 302호 등)
6. **교실 선택**: 공학관 501호 클릭하여 상세 정보 확인
7. **즐겨찾기 추가**: 정기적으로 사용할 예정이므로 즐겨찾기에 추가
8. **정보 공유**: 팀원들에게 장소 공유 (카카오톡, 문자 등)

**결과**: 
민수는 5분 이내에 조건에 맞는 교실을 찾았고, 팀원들과 안정적으로 회의를 진행할 수 있게 되었습니다.

### 시나리오 2: 공강 시간에 혼자 공부할 공간 찾기

**상황**: 
지은이는 매주 수요일 3교시(11시)가 공강입니다. 다음 수업은 5교시(1시)에 시작하므로, 2시간 동안 조용히 개인 공부를 할 수 있는 빈 교실을 찾고 있습니다.

**사용자 플로우**:

1. **앱 접속**: 수요일 아침 지은이가 앱을 실행
2. **검색 조건 입력**:
   - 요일: 수요일 선택
   - 시간: 3교시(11:00-12:00) 선택
3. **검색 실행**: 검색 버튼 클릭
4. **필터 적용**:
   - 건물 필터에서 "인문관" 선택 (다음 수업이 인문관에서 있음)
5. **결과 확인**: 인문관의 빈 교실 목록 표시
6. **사용 가능 시간 확인**: 각 교실별로 "12:50까지 사용 가능" 등의 정보 확인
7. **교실 선택**: 인문관 205호가 4교시까지 비어있어 여유롭게 사용 가능함을 확인
8. **즐겨찾기 추가**: 매주 같은 시간에 사용할 예정이므로 즐겨찾기에 추가

**다음 주 재사용**:
1. 다음 주 수요일, 지은이는 즐겨찾기 탭에서 인문관 205호를 바로 확인
2. 해당 시간에 여전히 비어있음을 확인하고 바로 이동

**결과**: 
지은이는 매주 공강 시간마다 교실을 찾아다니는 시간을 절약하고, 안정적으로 학습할 수 있는 공간을 확보했습니다.

### 시나리오 3: 동아리 정기 모임 장소 찾기

**상황**: 
동아리 회장 준호는 매주 금요일 저녁 6시에 15명의 동아리 회원들과 정기 모임을 합니다. 토론과 발표가 있어 넉넉한 공간과 빔프로젝터가 필요합니다.

**사용자 플로우**:

1. **웹사이트 접속**: 준호가 노트북으로 서비스 접속
2. **검색 조건 입력**:
   - 요일: 금요일 선택
   - 시간: 18:00-20:00 선택
3. **검색 실행**: 검색 버튼 클릭
4. **필터 적용**:
   - 시설 필터: "빔프로젝터 있음" 체크
   - 건물 필터: "경영관", "공학관" 선택 (접근성 좋은 중앙 건물들)
5. **결과 정렬**: 수용 인원 많은 순으로 정렬
6. **교실 비교**: 여러 교실의 상세 정보를 비교
   - 경영관 501호: 30명 수용, 빔프로젝터, U자형 배치
   - 공학관 401호: 40명 수용, 빔프로젝터, 계단식 배치
7. **교실 선택**: U자형 배치가 토론에 적합한 경영관 501호 선택
8. **즐겨찾기 추가**: 매주 같은 장소 사용을 위해 즐겨찾기 추가
9. **정보 기록**: 동아리 단체 채팅방에 장소 공지

**주간 사용**:
- 매주 금요일, 준호는 즐겨찾기에서 해당 교실이 여전히 비어있는지 빠르게 확인
- 만약 해당 시간에 사용 중이라면, 동일 조건으로 다시 검색하여 대체 교실 찾기

**결과**: 
준호는 동아리 모임 장소를 안정적으로 확보했고, 매주 장소를 찾는 스트레스에서 벗어났습니다.

## 5. 화면 구성 개념

> 참고: 실제 와이어프레임과 디자인 가이드는 별도 문서로 작성될 예정입니다.

### 5.1 메인 검색 화면

**구성 요소**:
- **상단**: 서비스 로고 및 제목
- **검색 섹션** (화면 중앙):
  - 요일 선택 버튼 (월~일, 가로 스크롤 또는 드롭다운)
  - 시간 선택 영역
    - 교시제 탭: 1교시, 2교시... 버튼 형태
    - 시간대 탭: 시작 시간과 종료 시간 선택 (타임 피커)
  - 검색 버튼 (큰 버튼으로 강조)
- **하단 네비게이션**:
  - 검색 (홈)
  - 즐겨찾기
  - 설정/정보

**인터랙션**:
- 요일과 시간을 선택하면 검색 버튼 활성화
- 검색 버튼 클릭 시 결과 화면으로 전환
- 즐겨찾기에서 최근 사용 교실 미리보기 (2~3개)

---

### 5.2 검색 결과 화면

**구성 요소**:
- **상단 헤더**:
  - 뒤로 가기 버튼
  - 검색한 조건 요약 표시 (예: "수요일 3교시")
  - 필터 버튼 (아이콘)
- **필터 패널** (토글 형태 또는 모달):
  - 건물별 체크박스
  - 시설별 체크박스
  - 교실 유형별 체크박스
  - 적용하기 버튼
- **정렬 옵션**:
  - 건물별
  - 수용인원순
  - 교실 번호순
- **결과 리스트**:
  - 각 교실 카드 형태로 표시
  - 교실명 (예: 공학관 301호)
  - 건물 및 층 정보
  - 수용 인원 아이콘 + 숫자
  - 시설 아이콘 (빔프로젝터, 컴퓨터 등)
  - 사용 가능 시간 (예: "14:50까지 사용 가능")
  - 즐겨찾기 아이콘 (별 모양)
- **빈 상태**: 
  - 결과가 없을 경우 안내 메시지
  - 필터 조건 완화 제안

**인터랙션**:
- 카드 클릭 시 교실 상세 화면으로 이동
- 별 아이콘 클릭 시 즐겨찾기 추가/제거
- 필터 적용 시 리스트 실시간 업데이트
- 무한 스크롤 또는 페이지네이션

---

### 5.3 교실 상세 정보 화면

**구성 요소**:
- **상단 헤더**:
  - 뒤로 가기 버튼
  - 교실명 (예: 공학관 301호)
  - 즐겨찾기 버튼
- **교실 기본 정보**:
  - 건물명 및 층수
  - 교실 유형 (강의실, 실습실 등)
  - 수용 인원
- **시설 정보**:
  - 보유 시설 목록 (아이콘 + 텍스트)
  - 좌석 배치 형태 설명
- **시간 정보**:
  - 선택한 시간대 사용 가능 여부 강조
  - "14:50까지 사용 가능" 등의 명확한 메시지
- **위치 안내**:
  - 층 및 위치 텍스트 설명
  - 주변 편의시설 (화장실, 엘리베이터 등)
- **교실 사진** (선택사항):
  - 실제 교실 사진 1~2장

**인터랙션**:
- 즐겨찾기 버튼으로 저장/해제
- 공유 버튼으로 친구들에게 교실 정보 공유 (선택사항)

---

### 5.4 즐겨찾기 화면

**구성 요소**:
- **상단 헤더**:
  - "즐겨찾기" 제목
  - 편집 버튼 (순서 변경, 삭제)
- **교실 리스트**:
  - 즐겨찾기한 교실 카드들
  - 교실명, 건물, 기본 정보 표시
  - 각 교실별 "현재 상태" 표시 (사용 중 / 비어 있음)
- **빈 상태**:
  - 즐겨찾기가 없을 경우 안내 메시지
  - "교실 검색하러 가기" 버튼

**인터랙션**:
- 카드 클릭 시 해당 교실 상세 화면으로 이동
- 편집 모드에서 드래그로 순서 변경
- 편집 모드에서 삭제 버튼으로 제거
- 카드에서 바로 검색 조건 설정하여 재검색 가능

### 5.5 설정/정보 화면 (선택사항)

**구성 요소**:
- 서비스 소개
- 사용 방법 가이드
- 데이터 업데이트 정보 (마지막 업데이트 날짜)
- 피드백/문의하기
- 이용 약관 및 개인정보 처리방침 (필요시)

## 6. 제약사항 및 가정

### 6.1 서비스 범위

- **단일 학교 대상**: 초기 버전은 특정 학교 하나만을 대상으로 함
- **학생 전용은 아님**: 로그인이 불필요하므로 누구나 접근 가능하지만, 주요 타겟은 재학생
- **공개 서비스**: 별도의 회원가입이나 인증 절차 없이 누구나 사용 가능

### 6.2 데이터 관련

- **데이터 수집 방법 미정**: 
  - 교실 정보와 시간표 데이터를 어떻게 수집할지는 추후 결정
  - 가능한 옵션: 수동 입력, 학교 시스템 정보 활용, 사용자 제보 등
- **데이터 업데이트 주기**: 
  - 학기 초 또는 시간표 변경 시 업데이트 필요
- **데이터 정확성**: 
  - 최대한 정확한 정보를 제공하되, 돌발 상황(휴강, 강의실 변경 등)은 반영 불가

### 6.3 운영 관련

- **관리자 기능**: 데이터를 관리할 수 있는 별도의 관리자 페이지 필요 (별도 개발)
- **피드백 수집**: 사용자 피드백을 받을 수 있는 채널 마련 (이메일, 폼 등)
- **유지보수**: 학기마다 데이터 갱신 필요

## 7. 향후 확장 계획 (MVP 이후)

MVP 출시 후 사용자 피드백을 바탕으로 다음 기능들을 단계적으로 추가할 수 있습니다:

### 7.1 단기 확장 기능 (Phase 2)

#### 실시간 사용 현황
- **목적**: 교실이 실제로 비어있는지 실시간으로 확인
- **방법**: 
  - 사용자 제보 기반 ("현재 이 교실 비어있음/사용 중" 버튼)
  - 학교와 협력하여 실시간 정보 제공
- **효과**: 시간표상 비어있지만 실제로 사용 중인 경우를 방지

#### 층별 지도 표시
- **목적**: 교실 위치를 시각적으로 파악
- **내용**: 건물별 층별 지도에 교실 위치 표시
- **효과**: 처음 가는 교실도 쉽게 찾을 수 있음

#### 사용자 리뷰/평가
- **목적**: 교실 환경에 대한 실제 사용자 의견 공유
- **내용**: 별점, 간단한 코멘트 (예: "조용함", "콘센트 많음", "추움")
- **효과**: 데이터에 없는 실질적인 정보 제공

### 7.2 중기 확장 기능 (Phase 3)

#### 교실 예약 기능
- **목적**: 빈 교실을 미리 예약하여 확보
- **조건**: 학교 측의 승인 및 시스템 연동 필요
- **기능**:
  - 예약 가능 교실 표시
  - 예약 신청 및 승인 프로세스
  - 예약 확인 및 취소
- **효과**: 교실 사용의 불확실성 완전히 해소

#### 회원가입 및 계정 기능
- **목적**: 여러 기기에서 즐겨찾기 동기화 및 개인화 기능
- **기능**:
  - 학교 이메일 인증 (재학생 확인)
  - 즐겨찾기 동기화
  - 예약 내역 관리
  - 개인화 추천 (자주 가는 건물 기반)

#### 그룹/팀 기능
- **목적**: 팀 단위로 교실 검색 및 일정 관리
- **기능**:
  - 팀원들의 시간표 통합하여 공통 공강 시간 찾기
  - 팀 즐겨찾기 공유
  - 팀 일정 캘린더

### 7.3 장기 확장 기능 (Phase 4)

#### 다중 학교 지원
- **목적**: 여러 대학교에서 서비스 사용 가능
- **구현**:
  - 학교 선택 기능
  - 학교별 데이터 분리 관리
  - 학교별 시간 체계 커스터마이징
- **효과**: 서비스 확장 및 더 많은 학생들에게 가치 제공

#### AI 추천 기능
- **목적**: 사용자 패턴을 학습하여 최적의 교실 추천
- **기능**:
  - 사용 기록 분석
  - 비슷한 사용자 패턴 분석
  - 맞춤형 교실 추천
- **효과**: 검색 시간 단축 및 만족도 향상

#### 커뮤니티 기능
- **목적**: 학생들 간 정보 공유 및 교류
- **기능**:
  - 스터디 그룹 모집 게시판
  - 교실별 채팅방
  - 공지사항 및 팁 공유
- **효과**: 단순 검색 도구를 넘어 커뮤니티 플랫폼으로 발전

#### 외부 서비스 연동
- **목적**: 학생들이 사용하는 다른 서비스와 연계
- **연동 대상**:
  - 학교 공식 앱/포털
  - 시간표 관리 앱
  - 캘린더 앱 (Google Calendar, Apple Calendar)
  - 지도 앱 (카카오맵, 네이버 지도)
- **효과**: 사용자 편의성 극대화

## 8. 성공 지표 (KPI)

MVP 출시 후 서비스의 성공을 측정하기 위한 지표:

### 8.1 사용자 지표
- **DAU/MAU** (일간/월간 활성 사용자 수)
- **신규 사용자 증가율**
- **재방문율**: 서비스를 다시 사용하는 사용자 비율
- **평균 세션 시간**: 한 번 방문 시 평균 사용 시간

### 8.2 기능 사용 지표
- **검색 완료율**: 검색을 시작한 사용자 중 결과까지 확인한 비율
- **필터 사용률**: 필터 기능을 사용하는 사용자 비율
- **즐겨찾기 사용률**: 즐겨찾기를 추가한 사용자 비율
- **평균 검색 횟수**: 사용자당 평균 검색 횟수

### 8.3 만족도 지표
- **사용자 만족도 설문**: 5점 척도 설문 (목표: 4.0 이상)
- **추천 의향**: NPS (Net Promoter Score)
- **피드백 내용 분석**: 긍정/부정 피드백 비율

### 8.4 비즈니스 지표
- **목표 달성률**: 
  - 출시 1개월 내 100명 이상 사용
  - 출시 3개월 내 500명 이상 사용
  - 재방문율 50% 이상
- **입소문 효과**: 자연 유입 사용자 비율

## 9. 리스크 및 대응 방안

### 9.1 데이터 관련 리스크

**리스크**: 
- 시간표 데이터 확보의 어려움
- 데이터 정확성 유지의 어려움
- 데이터 업데이트 지연

**대응 방안**:
- 학교 측과 협의하여 공식 데이터 제공 받기
- 초기에는 일부 건물만 대상으로 시작 (점진적 확대)
- 사용자 제보 기능으로 오류 수정
- 데이터 업데이트 필요 시 공지 및 안내

### 9.2 사용자 관련 리스크

**리스크**:
- 초기 사용자 확보 어려움
- 홍보 채널 부족
- 사용자 이탈

**대응 방안**:
- 학교 커뮤니티, SNS를 통한 입소문 마케팅
- 학생회, 동아리 등과 협력하여 홍보
- 초기 사용자 대상 피드백 적극 수렴 및 개선
- 사용 방법이 매우 간단하도록 UX 최적화

### 9.3 운영 관련 리스크

**리스크**:
- 관리 인력 부족
- 지속적인 유지보수 부담
- 예상치 못한 오류 발생

**대응 방안**:
- 효율적인 운영 프로세스 구축
- 사용자 피드백 채널 명확히 안내
- 문제 발생 시 신속 대응 체계 마련
- 초기에는 작은 규모로 시작하여 점진적 확대

### 9.4 경쟁 관련 리스크

**리스크**:
- 유사 서비스 출현
- 학교 공식 앱에서 유사 기능 제공

**대응 방안**:
- 빠른 MVP 출시로 선점 효과
- 사용자 경험을 최우선으로 차별화
- 지속적인 기능 개선 및 업데이트
- 필요시 학교 측에 공식 서비스로 제안

- **공강교실**: 특정 시간에 수업이 배정되지 않아 비어있는 강의실
- **교시**: 학교의 수업 시간 단위 (보통 50분 또는 1시간)
- **MVP** (Minimum Viable Product): 최소 기능 제품, 핵심 기능만 포함한 초기 버전
- **필터링**: 특정 조건에 맞는 결과만 선별하는 기능
- **즐겨찾기**: 자주 사용하는 항목을 저장하여 빠르게 접근하는 기능
- **페르소나**: 서비스의 대표적인 사용자 유형을 가상으로 정의한 것
- **사용자 시나리오**: 사용자가 서비스를 사용하는 구체적인 상황과 흐름
